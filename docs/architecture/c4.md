# C4 Architecture

This document shows the current platform architecture using C4-style diagrams.

## System Context

```mermaid
C4Context
    title Keycloak Admin Platform - System Context

    Person(admin, "Administrator", "Uses REST APIs to manage users, groups, roles and user profile attributes.")
    Person(app, "Application Backend", "Can call the admin API or publish notification commands when trusted.")

    System_Boundary(platform, "Keycloak Admin Platform") {
        System(adminApi, "backend-keycloak", "Generic REST API for Keycloak realm administration.")
        System(auditService, "backend-audit", "Persists identity and notification audit events.")
        System(notificationService, "backend-notification", "Sends generic e-mails and emits notification result events.")
    }

    System_Ext(keycloak, "Keycloak", "Identity provider and administration target.")
    System_Ext(kafka, "Kafka", "Event and command broker.")
    System_Ext(smtp, "SMTP Server", "E-mail delivery provider.")
    SystemDb_Ext(auditDb, "Audit PostgreSQL", "Audit event storage.")
    SystemDb_Ext(notificationDb, "Notification PostgreSQL", "Notification idempotency and outbox storage.")

    Rel(admin, adminApi, "Manages identity data", "HTTP/JSON")
    Rel(app, adminApi, "Calls identity administration endpoints", "HTTP/JSON")
    Rel(app, kafka, "May publish notification commands (optional via profile)", "Kafka / Optional")
    Rel(adminApi, keycloak, "Manages realm resources", "Keycloak Admin API")
    Rel(adminApi, kafka, "Publishes identity events and notification commands when enabled", "Kafka")
    Rel(kafka, auditService, "Delivers identity.events and notification.events", "Kafka")
    Rel(kafka, notificationService, "Delivers notification.commands", "Kafka")
    Rel(notificationService, kafka, "Publishes notification.events", "Kafka")
    Rel(notificationService, smtp, "Sends e-mails", "SMTP")
    Rel(auditService, auditDb, "Stores audit events", "JDBC")
    Rel(notificationService, notificationDb, "Stores command state and outbox rows", "JDBC")
```

## Container View

```mermaid
C4Container
    title Keycloak Admin Platform - Container View

    Person(admin, "Administrator")

    System_Ext(keycloak, "Keycloak")
    System_Ext(smtp, "SMTP Server")

    Container_Boundary(platform, "Keycloak Admin Platform") {
        Container(adminApi, "backend-keycloak", "Quarkus / Java 21", "REST API for Keycloak realm administration. Can run with Kafka disabled or enabled.")
        Container(auditService, "backend-audit", "Quarkus / Java 21", "Kafka consumer and audit query API.")
        Container(notificationService, "backend-notification", "Quarkus / Java 21", "Kafka/REST notification service with PostgreSQL-backed e-mail outbox.")
        System_Ext(kafka, "Kafka (optional)", "Event broker enabled only with 'kafka' Quarkus profile")
        ContainerDb(auditDb, "Audit PostgreSQL", "PostgreSQL", "Stores immutable audit events.")
        ContainerDb(notificationDb, "Notification PostgreSQL", "PostgreSQL", "Stores processed command ids and notification_email_outbox.")
    }

    Rel(admin, adminApi, "Uses", "HTTP/JSON")
    Rel(adminApi, keycloak, "Reads and writes realm data", "Keycloak Admin API")
    Rel(adminApi, kafka, "Publishes identity.events and notification.commands ((when enabled))", "Kafka")
    Rel(kafka, auditService, "Consumes identity.events and notification.events (when enabled)", "Kafka")
    Rel(kafka, notificationService, "Consumes notification.commands (when enabled)", "Kafka")
    Rel(notificationService, kafka, "Publishes notification.email.sent / notification.email.failed (when enabled)", "Kafka")
    Rel(notificationService, smtp, "Sends e-mails", "SMTP")
    Rel(auditService, auditDb, "Reads/writes", "JDBC")
    Rel(notificationService, notificationDb, "Reads/writes idempotency and outbox state", "JDBC")
```

## Kafka Enabled Mode

When `IDENTITY_EVENTS_ENABLED=true`, mutating identity operations publish
`identity.events` after Keycloak succeeds. If publication fails, the HTTP
request fails.

When `NOTIFICATION_COMMANDS_ENABLED=true`, the update-password e-mail action
publishes `notification.commands` instead of using Keycloak SMTP. The
notification service persists the command and an outbox row transactionally,
then a scheduled worker sends SMTP and publishes `notification.email.sent` or
`notification.email.failed`.
